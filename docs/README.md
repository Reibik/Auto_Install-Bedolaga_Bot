## Обзор

ST Village — это автоматизированная панель управления для развертывания и управления стеком:
- **Bedolaga Telegram Bot** — backend бота
- **Bedolaga Cabinet** — веб-кабинет пользователя
- **Caddy** — reverse proxy с автоматическим SSL

## Системные требования

| Компонент | Минимум | Рекомендуется |
|-----------|---------|---------------|
| **ОС** | Ubuntu 20.04+ / Debian 11+ | Ubuntu 22.04 LTS |
| **RAM** | 1 GB | 2 GB |
| **Диск** | 5 GB свободно | 10 GB |
| **Права** | root | root |
| **Сеть** | Открытые порты 80 и 443 | Статический IP |
| **Домены** | 2 домена с A-записями | С настроенным DNS |

> Docker и Docker Compose устанавливаются автоматически, если не обнаружены.

## Быстрая установка

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/Reibik/Auto_Install-Bedolaga_Bot/main/st_village.sh)
```

## Архитектура

```
Браузер → Caddy (80/443, auto-SSL)
              ├─ bot.example.com     → remnawave_bot:8080      (Backend API)
              └─ cabinet.example.com
                    ├─ /api/*        → remnawave_bot:8080      (Cabinet API, strip /api)
                    └─ /*            → cabinet_frontend:80      (React SPA)
```

Все контейнеры объединены в Docker-сеть `remnawave-network`.

## Возможности

### Основные

- Автоматическая установка Docker и зависимостей
- Клонирование репозиториев бота и кабинета
- Генерация конфигов Caddy, docker-compose, JWT-секретов
- Интерактивные мастера настройки `.env` с валидацией

### Мониторинг

- **Health Check** — автоматическая проверка сервисов каждые 10 мин. с Telegram-алертами
- **Watchdog** — авто-рестарт упавших контейнеров каждые 5 мин.
- **DNS-валидация** перед запуском Caddy
- **Логирование** всех операций в файл с ротацией

### Диагностика

- Полная проверка: контейнеры, HTTP, DNS, порты
- Проверка SSL-сертификатов
- Мониторинг ресурсов контейнеров (CPU, RAM, сеть)
- Валидация `.env` конфигурации

### Безопасность

- UFW файрвол (порты 22, 80, 443)
- Fail2ban (защита SSH от брутфорса)
- Telegram-уведомления о событиях

### Обслуживание

- Бэкапы (ручные и автоматические) с ротацией и восстановлением
- Миграция между серверами (экспорт/импорт)
- Управление Swap, очистка Docker
- Самообновление скрипта
