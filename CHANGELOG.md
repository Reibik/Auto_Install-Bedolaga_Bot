# Changelog

Все значимые изменения в проекте ST Village будут документированы в этом файле.

Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/1.0.0/),
и этот проект придерживается [Semantic Versioning](https://semver.org/lang/ru/).

## [18.0] - 2026-04-05

### Добавлено
- **Cron Health Check** — автоматическая проверка сервисов каждые 10 минут с Telegram-уведомлениями при проблемах (контейнеры, HTTPS, SSL)
- **Container Watchdog** — авто-рестарт упавших контейнеров каждые 5 минут с Telegram-уведомлениями
- **DNS-валидация перед запуском Caddy** — проверка A-записей доменов перед стартом, предотвращение проблем с SSL-сертификатами
- **Логирование в файл** — все операции записываются в `st_village.log` с автоматической ротацией при 10 МБ
- **Кэширование версий** — быстрая загрузка дашборда без `git fetch` (кэш 5 мин.), отдельная кнопка "Проверить обновления"
- **Просмотр лога панели** — новый пункт меню для просмотра последних записей лога
- `CONTRIBUTING.md` — руководство по участию в проекте
- `SECURITY.md` — политика безопасности и ответственного раскрытия
- `.github/workflows/security.yml` — Security Scan workflow (ShellCheck аудит, поиск секретов, проверка безопасности)

### Изменено
- Главное меню расширено: добавлен пункт "Проверить обновления компонентов" (11)
- Меню "Система и безопасность" расширено: добавлена секция "Мониторинг" (Health Check, Watchdog, Лог)
- `start_project()` теперь проверяет DNS перед запуском Caddy
- `main()` использует `case` вместо цепочки `if` для cron-аргументов
- `init_tty()` поддерживает `cron_watchdog` аргумент

### Исправлено (ShellCheck)
- SC2164: добавлен `|| exit 1` к `cd` в подоболочках
- SC2086: закавычены `${HTTP_PORT}` и `${HTTPS_PORT}` в `for` циклах
- SC2086: закавычен `return "$has_conflicts"`
- SC2116: заменён `echo | tr -d ' '` на parameter expansion `${var// /}`
- Устранены предсказуемые временные файлы (`/tmp/stv_*.$$` → `mktemp`)
- Устранены мёртвые записи переменных в подоболочках (check_versions)
