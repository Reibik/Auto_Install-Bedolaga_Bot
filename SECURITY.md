# Политика безопасности

## Поддерживаемые версии

| Версия | Поддержка        |
|--------|------------------|
| 18.x   | :white_check_mark: Активная поддержка |
| 17.x   | :warning: Только критические исправления |
| < 17.0 | :x: Не поддерживается |

## Сообщить об уязвимости

Если вы обнаружили уязвимость в проекте, **НЕ** создавайте публичный Issue.

### Как сообщить

1. Напишите в [Telegram-чат поддержки](https://t.me/+wTdMtSWq8YdmZmVi) с пометкой **[SECURITY]**
2. Или создайте [приватный отчёт об уязвимости](https://github.com/Reibik/Auto_Install-Bedolaga_Bot/security/advisories/new) через GitHub Security Advisory

### Что включить в отчёт

- Описание уязвимости
- Шаги для воспроизведения
- Потенциальное влияние
- Предложенное исправление (если есть)

### Время реагирования

- **Подтверждение получения:** в течение 48 часов
- **Первичная оценка:** в течение 7 дней
- **Исправление:** зависит от серьёзности, обычно 14-30 дней

## Рекомендации по безопасности

### Для пользователей скрипта

1. **Запускайте только от root** — скрипт требует root-привилегий для управления Docker и системными сервисами
2. **Используйте актуальную версию** — обновляйте скрипт через встроенное меню самообновления
3. **Защитите сервер:**
   - Используйте SSH-ключи вместо паролей
   - Включите UFW файрвол (доступно через меню скрипта)
   - Включите Fail2ban (доступно через меню скрипта)
4. **Храните `.env` файлы в безопасности** — они содержат токены и секреты
5. **Регулярно делайте бэкапы** — используйте встроенную функцию автоматических бэкапов

### Для контрибуторов

1. **Не хардкодьте секреты** — используйте переменные окружения и `.env` файлы
2. **Валидируйте пользовательский ввод** — особенно домены, порты, пути
3. **Используйте безопасные временные файлы** — `mktemp` вместо предсказуемых имён
4. **Кавычьте переменные** — предотвращение инъекций через glob/word splitting
5. **Минимальные привилегии** — не запускайте от root то, что может работать без него

## Известные ограничения

- Скрипт предназначен для запуска на выделенных серверах, а не на shared-хостинге
- Docker-контейнеры работают с привилегиями по умолчанию (для доступа к портам 80/443)
- JWT-секреты генерируются автоматически через `openssl rand` при установке
