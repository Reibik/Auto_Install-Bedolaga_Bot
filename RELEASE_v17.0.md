## ✨ Новые возможности

### 🔌 Интеллектуальная проверка портов
- **Автоматическая проверка** занятости портов 80 и 443 перед установкой
- **Детальная информация** о конфликтующих процессах
- **Интерактивные подсказки** по решению конфликтов
- **Возможность продолжить** установку или отменить операцию

```bash
# Пример вывода:
🔌 ПРОВЕРКА ПОРТОВ ПЕРЕД УСТАНОВКОЙ
----------------------------------------
  Порт 80: ✓ свободен
  Порт 443: ✗ занят (nginx)

Рекомендации:
  1. Остановите конфликтующие сервисы
  2. Или используйте другие порты через переменные окружения:
     ST_VILLAGE_HTTP_PORT=8080 ST_VILLAGE_HTTPS_PORT=8443 ./st_village.sh
```

### ⚙️ Конфигурируемые пути установки
Теперь можно настроить пути установки через переменные окружения:

```bash
# Изменить базовую директорию (по умолчанию /root)
export ST_VILLAGE_BASE_DIR=/opt/st_village

# Изменить порты (по умолчанию 80 и 443)
export ST_VILLAGE_HTTP_PORT=8080
export ST_VILLAGE_HTTPS_PORT=8443

# Запустить установку
./st_village.sh
```

## 📥 Установка

### Быстрая установка (рекомендуется)

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/Reibik/Auto_Install-Bedolaga_Bot/main/st_village.sh)
```

### Установка с пользовательскими настройками

```bash
# Скачать скрипт
curl -fsSL https://raw.githubusercontent.com/Reibik/Auto_Install-Bedolaga_Bot/main/st_village.sh -o st_village.sh
chmod +x st_village.sh

# Настроить переменные окружения (опционально)
export ST_VILLAGE_BASE_DIR=/opt/myproject
export ST_VILLAGE_HTTP_PORT=8080
export ST_VILLAGE_HTTPS_PORT=8443

# Запустить установку
./st_village.sh
```

## 🔄 Обновление с v16.0

Если у вас уже установлена версия 16.0, обновление очень простое:

### Способ 1: Через меню панели (рекомендуется)

```bash
./st_village.sh
# Выберите: 9 → 11 (Самообновление скрипта)
```

### Способ 2: Вручную

```bash
curl -fsSL https://raw.githubusercontent.com/Reibik/Auto_Install-Bedolaga_Bot/main/st_village.sh -o /root/st_village.sh
chmod +x /root/st_village.sh
./st_village.sh
```

> **Примечание:** Все ваши конфигурации и данные сохранятся при обновлении.

## 🏗️ Архитектура проекта

```
Internet
    ↓
Caddy (ports 80/443, auto-SSL)
    ├─→ bot.example.com → remnawave_bot:8080 (Backend API)
    └─→ cabinet.example.com
           ├─→ /api/* → remnawave_bot:8080 (Cabinet API)
           └─→ /* → cabinet_frontend:80 (React SPA)
```

### Структура файлов на сервере

```
/root/  (или $ST_VILLAGE_BASE_DIR)
├── bot/                        # Bedolaga Bot
│   ├── docker-compose.local.yml
│   └── .env
├── cabinet/                    # Bedolaga Cabinet
│   ├── docker-compose.yml
│   ├── docker-compose.override.yml
│   └── .env
├── caddy/                      # Caddy reverse proxy
│   ├── docker-compose.yml
│   └── Caddyfile
├── backups/                    # Резервные копии
└── st_village.sh               # Панель управления
```
